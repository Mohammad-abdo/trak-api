Driver socket test page

This page connects to your Socket.IO server and listens for the driver wake-up events new-ride-available and new_ride_request (room driver-{id}). It also includes a dev-only API call to emit a fake event to a driver room.
idle
If you use a custom SOCKET_PATH in env, set it above. If SOCKET_ENFORCE_AUTH=1, you must provide a valid driver JWT. 

          

            The emit button calls POST /api/test/socket/emit-driver-ride (disabled in production).
